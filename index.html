Part 1: The Anatomy of a URL

A Uniform Resource Locator (URL) is more than just a name; it's a precise instruction set for your browser. It tells the browser *what* resource to get and *how* to get it.

  • Protocol (e.g., `https://`): The rules of communication. HTTPS (Hypertext Transfer Protocol Secure) ensures encrypted communication, essential for security.
  • Domain/Host (`www.foursix46.com`): The human-readable name of the server. This is the part that must be translated into an IP address.
  • Path (`/blog/url-explained`): Specifies the exact location of the file or resource on that server. It directs the server to the content you want to view.
  • Query Parameters (`?ref=social`): Optional data passed to the server, often used for tracking or filtering content. These start with a `?` followed by key-value pairs (`key=value`).
  • Fragment (`#title`): An optional internal marker. It tells the browser to scroll to a specific element *after* the page loads (the server never sees this part).

Part 2: The 5-Step Resolution Process

1. DNS Lookup: The Internet's GPS

The browser uses the Domain (e.g., `foursix46.com`) to find its unique IP Address (e.g., `192.168.1.1`) via a Domain Name System (DNS) server. This is the crucial translation step. [Image of DNS lookup process]

2. TCP/TLS Handshake: Securing the Line

Using the IP address, the browser initiates a **TCP Handshake** (SYN, SYN-ACK, ACK) to establish a reliable connection. For HTTPS, a subsequent **TLS Handshake** occurs to encrypt the entire session.

3. HTTP Request: Asking for the File

The browser sends a `GET` request (including the Path and any Query Parameters) to the server, essentially saying, "Please send me the file located at this Path."

4. Server Response: Sending the Code

The server finds the resource, prepares the HTML, and sends back an **HTTP Response**. This includes a status code (like `200 OK` or `404 Not Found`) and the payload of code (HTML, CSS, JavaScript).

5. Rendering: Building the Page

The browser takes the raw code, builds the structure (**DOM Tree**), applies the styles (**CSSOM Tree**), and finally "paints" the completed page onto your screen, resulting in the final view.
